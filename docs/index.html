TypeScript Zoho Payments Node ≥ 18 Apache-2.0

zohopay SDK

Typed TypeScript SDK for the Zoho Payments API. Covers OAuth 2.0, payment links, payment sessions, payments, customers, and refunds across India and US editions. Zero runtime dependencies — uses native fetch.

What's included

  • OAuth 2.0 — auth URL, code exchange, token refresh, revoke
  • Payment Links — create, update, cancel, list, get
  • Payment Sessions — create, get
  • Payments — create (US), get, list, verify
  • Customers — create, get, list (US), delete (US)
  • Refunds — create, get, list
  • Pluggable HTTP transport
  • Typed exception hierarchy

Source layout

  • src/auth/ — OAuth helpers & token lifecycle
  • src/net/ — transport & authenticated HTTP client
  • src/services/ — API resource services
  • src/models/ — response model types
  • src/params/ — request serializers & validators
  • src/exceptions.ts — exception hierarchy
  • examples/server.ts — local dev proxy server

Create a client

import { ZohoPayments, Edition } from "zohopay";

const client = ZohoPayments.builder()
  .accountId("YOUR_ACCOUNT_ID")
  .edition(Edition.IN)           // Edition.IN | Edition.IN_SANDBOX | Edition.US
  .accessToken("1000.xxxx.yyyy")
  .requestTimeoutMs(45_000)      // optional — default 60 s
  .build();
The builder is single-use. Calling build() twice throws. Use client.close() or await using client = … to release resources.

OAuth 2.0 flow

1 — Build the authorization URL

import { buildAuthorizationUrl, Edition, OAUTH_SCOPES } from "zohopay";

const authUrl = buildAuthorizationUrl({
  clientId:    "1000.YOUR_CLIENT_ID",
  accountId:   "YOUR_ACCOUNT_ID",
  redirectUri: "https://yourapp.example.com/oauth/callback",
  edition:     Edition.IN,
  accessType:  "offline",   // "offline" to receive a refresh token
  state:       "csrf-token-here",  // validate on callback to prevent CSRF
  scopes: [
    OAUTH_SCOPES.PAYMENT_LINKS_CREATE,
    OAUTH_SCOPES.PAYMENT_LINKS_READ,
    OAUTH_SCOPES.PAYMENTS_READ,
    OAUTH_SCOPES.CUSTOMERS_CREATE,
    OAUTH_SCOPES.REFUNDS_CREATE,
    OAUTH_SCOPES.REFUNDS_READ,
  ],
});
// Redirect the user to authUrl

2 — Parse callback & exchange code

import { parseOAuthCallback, exchangeCodeForToken } from "zohopay";

const callback = parseOAuthCallback(req.url);
// Validate callback.state against your stored CSRF token here

const tokenSet = await exchangeCodeForToken({
  code:         callback.code!,
  clientId:     "1000.YOUR_CLIENT_ID",
  clientSecret: "YOUR_CLIENT_SECRET",
  redirectUri:  "https://yourapp.example.com/oauth/callback",
  edition:      Edition.IN,
});
// tokenSet.accessToken  — use immediately
// tokenSet.refreshToken — store securely
// tokenSet.expiresIn    — seconds until expiry

3 — Refresh the access token

// The SDK does NOT auto-refresh. Call this when the token expires.
const fresh = await ZohoPayments.generateAccessToken({
  refreshToken: "1000.stored_refresh_token",
  clientId:     "1000.YOUR_CLIENT_ID",
  clientSecret: "YOUR_CLIENT_SECRET",
  redirectUri:  "https://yourapp.example.com/oauth/callback",
  edition:      Edition.IN,
});

client.updateOAuthToken(fresh);  // no rebuild needed
// fresh.isExpired() — true if expired (with 30 s buffer)

4 — Revoke a refresh token

await ZohoPayments.revokeToken({
  token:   "1000.stored_refresh_token",
  edition: Edition.IN,
});

Error handling

import {
  AuthenticationException,   // 401
  PermissionException,        // 403
  ResourceNotFoundException,  // 404
  InvalidRequestException,    // 400 / 422
  RateLimitException,         // 429
  ZohoPaymentsAPIException,   // any other non-2xx
  ConnectionException,        // network / timeout
} from "zohopay";

try {
  await client.payments().get("id");
} catch (err) {
  if (err instanceof AuthenticationException) {
    // Refresh the token and retry
  } else if (err instanceof RateLimitException) {
    // Back off and retry
  } else if (err instanceof InvalidRequestException) {
    console.error(err.codeString, err.apiErrorMessage);
  } else if (err instanceof ZohoPaymentsAPIException) {
    console.error(err.httpStatusCode, err.message, err.rawBody);
  } else if (err instanceof ConnectionException) {
    // Network error or request timeout
  }
}

Custom HTTP transport

Implement HttpTransport to add retries, proxies, or logging.

import { DefaultHttpTransport, ZohoPayments, Edition } from "zohopay";
import type { HttpTransport, ZohoRequest, ZohoResponse } from "zohopay";

class LoggingTransport implements HttpTransport {
  constructor(private readonly inner: HttpTransport) {}

  async execute(req: ZohoRequest): Promise<ZohoResponse> {
    console.log(`→ ${req.method} ${req.url}`);
    const res = await this.inner.execute(req);
    console.log(`← ${res.statusCode}`);
    return res;
  }
}

const client = ZohoPayments.builder()
  .accountId("YOUR_ACCOUNT_ID")
  .edition(Edition.IN)
  .accessToken("1000.xxxx.yyyy")
  .transport(new LoggingTransport(new DefaultHttpTransport()))
  .build();
transport() and requestTimeoutMs() are mutually exclusive — passing both throws at build time.

Dev proxy server

A local Express server proxies all Zoho API calls server-side, avoiding CORS issues during development.

bun run playground   # starts on http://127.0.0.1:3579
# Set PORT env var to override the port
The server binds to 127.0.0.1 only and is not accessible on the LAN.
